# 微信公众号内容转换规则参考

## 0. 内容保留原则

**必须保留：**
- ✅ 文章的核心观点和论据
- ✅ 数据、案例、引用
- ✅ 作者的独特见解
- ✅ 专业术语和概念

**可以调整：**
- ⚠️ 段落长度和结构
- ⚠️ 标题表达方式
- ⚠️ 开头和结尾格式
- ⚠️ 视觉元素呈现

**必须删除：**
- ❌ 空的序号行（markdown 解析问题导致）
- ❌ 无意义的空行

## 1. 标题优化

### 一级标题（文章标题）

**公众号标题特点：**
- 实用性强：「5个技巧让...」「如何...」
- 数字吸引：「3种方法」「10年经验」
- 情感共鸣：「终于...」「为什么我不...」
- 制造好奇：「你可能不知道...」「大多数人都...」

**优化方向：**
1. **增加具体性** - 「时间管理」→「5个时间管理技巧，让你每天多出2小时」
2. **添加数字** - 「写作技巧」→「10个写作技巧，让文章阅读量翻倍」
3. **突出价值** - 「Python入门」→「零基础学Python，30天从入门到实战」
4. **制造紧迫感** - 「投资建议」→「2024年投资建议：这3个机会不容错过」

**注意：** 避免过度标题党，确保标题与内容相符

### 二级标题（章节标题）

**推荐格式：**
- **数字编号**：「一、」「二、」「三、」
- **疑问式**：「为什么...？」「如何...？」「...是什么？」
- **陈述式**：「...的核心要点」「...的关键步骤」

## 2. 开头格式

### 推荐开头方式

**方式1: 痛点引入**
```
你是否也遇到过这样的困扰...
（描述用户痛点）
本文将帮你解决...
```

**方式2: 数据引入**
```
根据最新数据显示，...
（引用相关数据）
这表明...值得关注
```

**方式3: 场景引入**
```
想象这样一个场景...
（描述具体场景）
这正是...的核心问题
```

**方式4: 结论先行**
```
经过...实践，我总结了...
直接给出核心结论
```

**开头长度：** 不超过全文的10%

## 3. 正文结构

### 段落规范

- ✅ 每段不超过5行（约150字）
- ✅ 一段一个核心观点
- ✅ 段落间用空行分隔
- ✅ 避免大段文字堆砌

### 小标题使用

**什么时候用小标题：**
- 超过800字的文章
- 有多个并列要点
- 需要引导阅读节奏

**小标题频率：** 每300-500字一个

## 4. 视觉元素

### 图片使用

**推荐位置：**
- 开头后1-2段（吸引注意力）
- 中间每隔3-4段（缓解视觉疲劳）
- 重要概念后（增强理解）

**图片要求：**
- 宽度统一（建议900px）
- 清晰度高（分辨率≥1080px）
- 与内容相关
- 版权合规

### 表格使用

**适用场景：**
- 对比多个选项
- 展示数据
- 步骤流程
- 参数配置

### 分隔线使用

**推荐频率：** 每500-800字
**样式建议：*** 简单线条或主题色装饰线

## 5. 文字样式

### 字体颜色

**主题色：** 与品牌保持一致（如 #07c160）

**强调色：**
- 警示：#e74c3c（红色）
- 提示：#f39c12（橙色）
- 成功：#27ae60（绿色）
- 信息：#3498db（蓝色）

### 粗体使用

**使用场景：**
- 关键观点
- 重要数据
- 核心概念
- 行动号召

### 斜体使用

**使用场景：**
- 引用内容
- 专业术语首次出现
- 反语或强调

### 引用块使用

**使用场景：**
- 引用名言
- 用户评价
- 专家观点
- 特别说明

## 6. 结尾格式

### 推荐结尾方式

**方式1: 总结回顾**
```
本文我们讨论了...
（总结要点）
希望对你有所帮助
```

**方式2: 行动号召**
```
从今天开始，尝试...
（具体行动建议）
欢迎分享你的体验
```

**方式3: 互动引导**
```
你对...有什么看法？
（提出问题）
欢迎在评论区交流
```

**方式4: 展望未来**
```
随着...的发展，
（趋势展望）
期待与您共同成长
```

## 7. 特殊格式

### 列表使用

**无序列表：**
- 并列关系
- 没有优先级
- 补充说明

**有序列表：**
- 步骤流程
- 有先后顺序
- 程度递进

### 代码块使用

**适用场景：**
- 技术教程
- 配置示例
- 命令说明
- 代码演示

**样式要求：**
- 使用深色背景（#2d2d2d）
- 字号13px（移动端可读）
- 支持横向滚动
- 不强制换行

### 引用块使用

**适用场景：**
- 重要提示
- 注意事项
- 警告信息
- 特殊说明
